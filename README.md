Vaultr is a smol reading tracker built with vanilla HTML CSS and JS. it is client side which means all data only lives in the browsers "localStorage"

I am personally very passionate about reading so i thought it would be a good idea to do my summative on something i would actually want to use.So i made this literacy archive designed for tracking annual reading progress. You set a yearly goal, and the dashboard will show you your progress, repeated authors, your favorite genres and current active reads, in addition it will show you how many pages you need to be reading per day to finish your year on target . The deign is partly based on a more robust system like that, Goodreads. 

You can add, edit and delete books. 
You can mark them as finished or reading
The annual goal progress bar shows you the percentage and also the velocity
There are multiple animated cards on the dashboard 
You can search through your vault and you can sort by title, author, pages or date added. 
You can create new genres
It is also available in dark mode
you can import/export JSON for backup and restore 
And you can delete all data

Vaultr doesnt require any build steps or servers. You just open index.html in any browser lowk.

DATA STRUCTURE 

each book is stored as a plain js object. The full schema is 

```json
{
  "id":        "rec_4512",
  "title":     "Capital: Volume 1",
  "author":    "Karl Marx",
  "amount":    1152,
  "category":  "Theory",
  "status":    "Finished",
  "date":      "2026-01-10",
  "added":     "1/1/2026",
  "updatedAt": "2026-01-10T14:32:00.000Z"
}
```

The id is a unique record identifier. generated by generateUUID() using crypto.randomUUID(). The seed data uses a different format from the rest because of a change i made to the generation of the ID later on. 

The title is a string which must not begin or end with whitespace. validated by regex: ^\S(?:.*\S)?$.

The author is also a string validated by regex ^[A-Za-z]+(?:[ -][A-Za-z]+)*$ -only ASCII letters, spaces and hyphens are a okay. 

The amount is the number of pages, stored as a float via parseFloat. 

The ctageory is a genre among the genres array 

The status is one of two string literals, Finished or Reading. It is set at creation time and can be changed via inline editing. 

The date the book was finished on, in YYYY-MM-DD format. If the book is not finished this field says "in progress"

Added is the date the record was created. via new Date(). toLocaleDateString () output format is dependent on comp config. 

updatedAt is a string of an ISO8601 timestamp of the last mode. Set at creation and updated on every saveeEdit() whenever change is done haha. 

validation rules:

they are applied in addBook() before any record is created:
the Title must match ^\S(?:.*\S)?$ which means it cannot be empty or start with whitespace 
the Author must match ^[A-Za-z]+(?:[ -][A-Za-z]+)*$ -ASCII letters, spaces and hyphens only. Single word names are valid 

Pages must be greater than 0 

the Date required if mark finished is checked; must be a non empty string from date input. 

validation errors are displayed in a red .error-banner division above the form. Only one error is shown at a time, first check fail is shown.
